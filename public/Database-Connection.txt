DATABASE CONNECTION - FULL EXPLANATION
========================================

Yeh document batata hai ki website Firebase database se kaise connect hoti hai,
data kaise save hota hai, aur admin panel mein real-time kaise dikhta hai.


1. OVERVIEW
-----------
Website static hai (sirf HTML/CSS/JS), koi backend server nahi hai.
Browser directly Google ke Firebase servers se connect hota hai.
Firebase ek "Database as a Service" hai - Google manage karta hai, hume sirf
SDK use karna hota hai.


2. CONNECTION SETUP (3 steps)
------------------------------

Step 1: Firebase SDK load karna (registration.html / admin.html)
-----------------------------------------------------------------
HTML file mein yeh 3 script tags add kiye:
  <script src="firebase-app-compat.js"></script>    <- Firebase core
  <script src="firebase-firestore-compat.js"></script> <- Database module
  <script src="js/firebase-config.js"></script>     <- Hamari config

Pehle do scripts CDN se load hoti hain (Google ke servers se).
Teen number wali script local file hai.

Step 2: Firebase Config (js/firebase-config.js)
-------------------------------------------------
Yeh file mein Firebase project ki details hoti hain:
  var firebaseConfig = {
    apiKey: "AIzaSyBrXRJyJuqvQ5ikAOwoBthIojJSZ3KwyKY",
    authDomain: "cdlu-university.firebaseapp.com",
    projectId: "cdlu-university",
    storageBucket: "cdlu-university.firebasestorage.app",
    messagingSenderId: "336912868141",
    appId: "1:336912868141:web:4c8fa6e630bb44c27229a4"
  };

Yeh config aapne Firebase Console se copy ki thi:
  https://console.firebase.google.com/ -> Project Settings -> Your apps

Step 3: Initialize
-------------------
  firebase.initializeApp(firebaseConfig);   <- Firebase se handshake
  var db = firebase.firestore();            <- Database ka reference

initializeApp() function ek secure connection banata hai Firebase servers se.
Ye Google ke authentication system ka use karta hai (API key + project ID).


3. DATA SAVE KAISE HOTA HAI (registration.js)
-----------------------------------------------
Jab student form bhar ke SUBMIT karega:

  a) Form validation chalegi (required fields, phone pattern)
  b) Ek object banega:
       var reg = {
         id: "CDLU-1687945238417-A3F8",
         name: "Rahul Kumar",
         fatherName: "Suresh Kumar",
         dob: "2005-06-15",
         gender: "Male",
         category: "General",
         phone: "9876543210",
         programme: "BCA",
         qualification: "12th",
         percentage: "85%",
         registeredOn: "2026-06-27T10:30:00.000Z"
       };
  c) Local storage mein save:
       localStorage.setItem("cdluRegistrations", JSON.stringify(...));
       -> Yeh browser ke andar rehta hai (backup)

  d) Firebase mein save:
       db.collection("registrations").add(reg)
       -> Yeh HTTPS request bhejti hai Firebase server ko
       -> Server naya document banata hai
       -> Response aata hai (success ya error)
       -> Agar internet nahi hai to error aata hai, par localStorage safe hai

HTTP Request (background mein):
  Method: POST
  URL: https://firestore.googleapis.com/v1/projects/cdlu-university/...
  Headers:
    Authorization: Bearer <auto-generated-token>
    Content-Type: application/json
  Body: { fields: { name: {stringValue: "Rahul Kumar"}, ... } }


4. REAL-TIME READING (admin.js)
--------------------------------
Admin panel kholne par:

  let unsubscribe = db.collection("registrations")
    .orderBy("registeredOn", "desc")
    .onSnapshot(function(snapshot) {
      // Jab bhi data change hoga, yeh function chalega
      renderRegistrations();
    });

Kaam kaise karta hai:
  a) Pehli baar: Firebase se saara data aata hai (initial load)
  b) Ek WebSocket connection khulta hai (persistent connection)
  c) Firebase server par koi change hota hai to:
     - Server admin ke browser ko push notification bhejta hai
     - onSnapshot callback turant chal jata hai
     - Table update ho jati hai (bina page refresh kiye)

WebSocket:
  - Ye ek permanent 2-way connection hai browser aur server ke beech
  - HTTP ki tarah nahi ki har baar naya request bhejna pade
  - Isliye "real-time" hai - data aate hi turant dikhta hai
  - Koi polling nahi, koi interval nahi

6. SECURITY
------------
Firestore ki security rules abhi "test mode" mein hain:
  - Koi bhi padh sakta hai (read)
  - Koi bhi likh sakta hai (write)
  - Ye development ke liye theek hai

Production ke liye rules change karne chahiye:
  match /registrations/{doc} {
    allow read: if request.auth != null;   // Sirf logged in users
    allow create: if request.resource.data.name != null;
  }

Rules Firebase Console mein change hote hain:
  Build -> Firestore Database -> Rules tab


7. OFFLINE BEHAVIOR
--------------------
- Firebase SDK offline support built-in hai
- Agar internet nahi hai to data localStorage mein save ho jata hai
- Jab internet wapas aata hai to Firebase auto-sync kar deta hai
- Admin panel dono sources se data merge karta hai (localStorage + Firebase)


8. SUMMARY
-----------
| Step | Kya Hota Hai | Kaise |
|------|--------------|-------|
| 1 | Firebase SDK load | CDN se script tags |
| 2 | Config set | API key, project ID, etc. |
| 3 | Initialize | firebase.initializeApp() |
| 4 | Save data | db.collection().add() |
| 5 | Read data | db.collection().onSnapshot() |
| 6 | Real-time update | WebSocket push notification |
| 7 | Offline backup | localStorage |

Koi bhi device, koi bhi browser, koi bhi location - sabhi ek hi database se
connected hain. Isliye ek student mobile se registration karega to admin
laptop par turant dekh sakta hai.
